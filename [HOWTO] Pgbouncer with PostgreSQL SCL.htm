[HOWTO] Pgbouncer with PostgreSQL SCL

HOWTO use Pgbouncer with PostgreSQL (SCL)
# PGBOUNCER : 1.7.2
# POSTGRESQL: 9.5 SCL

PGBOUNCER have to be installed on each application server.

Prerequisite

Following packages have to be installed. (Without, configure will fail)
  • openssl-devel
  • libevent-devel
  • python-psycopg2.x86_64
  • rh-postgresql95

Each application server have to :

  • have a postgres user.
  • a /produits/pgsql filesystem

These can be asked by HPCSA.

SSH key of postgres user has to be exchange between all database servers and application servers.

Each database must have a password of postgres user setted. (For auth_file) 

Installation


Download PgBouncer  source from https://pgbouncer.github.io/downloads/

 

As postgres user, extract pgbouncer-1.7.2.tar.gz

tar xzvf pgbouncer-1.7.2.tar.gz

 

Then compile package.

cd /produits/pgsql/pgbouncer-1.7.2

./configure

then make

Copy binaries

ln -s /produits/pgsql/pgbouncer-1.7.2 /produits/pgsql/pgbouncer/
cp pgbouncer /produits/pgsql/pgbouncer/
cp etc/mkauth.py /produits/pgsql/pgbouncer/
cp etc/pgbouncer.ini /produits/pgsql/pgbouncer/

WARNING: 

Each database must have its own pgbouncer ini file and user file. 


create user file

cd /produits/pgsql/pgbouncer/
./mkauth.py [dbname]_userlist.txt "user=postgres port=[portnum] host=[dbserver] password=[db_postgres_password]"
A file pgbouncer_userlist.txt will be created

 

Now, configure pgbouncer.ini

WARNING : range port for pgbouncer should be between 12420 and 12429.


vi [dbname]_pgbouncer.ini

Add these lines and modify necessary information.

[databases]
[appl_name] = host=[primary_svr] port=[port] dbname=[db_name]
[pgbouncer]
syslog = 1
pidfile = /produits/pgsql/pgbouncer/pgbouncer.pid
listen_addr = *
listen_port = 124[20-29]
auth_type = md5
auth_file = /produits/pgsql/pgbouncer/[dbname]_userlist.txt
pool_mode = session
admin_users = postgres
server_reset_query = DISCARD ALL
max_client_conn = 100
default_pool_size = 20

create .profile file :

$ cat .profile

. /opt/rh/rh-postgresql95/enable

export PATH=/produits/pgsql/pgbouncer/:$PATH

 

and :

 

$ . .profile 


Onve pgbouncer.ini finished,  start pgbouncer
pgbouncer -d /produits/pgsql/pgbouncer/[dbname]_pgbouncer.ini
2017-06-01 11:23:21.718 58945 LOG File descriptor limit: 1024 (H:4096), max_client_conn: 100, max fds possible: 150

 

Configure automatic failover

Each database sever will update application server [dbname]_pgbouncer.ini file once a failover is initiated.

For this, each database server has to have a template of [dbname]_pgbouncer.ini in /bases/pgsql/[db_name]/admin that have to be deployed on each application server if it became primary after a failover.

Then create the script failover.sh in /bases/pgsql/[db_name]/admin  with the  following content on each database server.

 #!/bin/sh

. /produits/pgsql/env/env_[dbname]_hot0

repmgr standby promote -f ${BASE}/admin/repmgr.conf

export dbname=[dbname]

for appsvr in [svr1] [svr2] ...
do
rsync --delete -a -k --perms  $BASE/admin/${dbname}_pgbouncer.ini ${appsvr}:/produits/pgsql/pgbouncer/
ssh -oStrictHostKeyChecking=no ${appsvr} /produits/pgsql/pgbouncer/pgbouncer -R -d /produits/pgsql/pgbouncer/${dbname}_pgbouncer.ini
done

Then, modify repmgr.conf on each database server

cluster=heaateaa
node=1
node_name=heaateaa_hot0
conninfo='host=heaalx11 port=12400 user=repmgr dbname=repmgr'
master_response_timeout=900
reconnect_attempts=5
reconnect_interval=1
pg_bindir=/opt/rh/rh-postgresql94/root/usr/bin
remote_command_prefix=scl enable rh-postgresql94 --
use_replication_slots = 1
failover=automatic
logfile=/bases/pgsql/heaateaa_hot0/var/log/repmgr.log
promote_command='/bases/pgsql/heaateaa_hot0/admin/failover.sh'
follow_command='repmgr standby follow -f /bases/pgsql/heaateaa_hot0/admin/repmgr.conf --log-to-file'
