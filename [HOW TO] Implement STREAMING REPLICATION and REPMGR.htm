[HOW TO] Implement STREAMING REPLICATION and REPMGR
 

[HOW TO] Implement STREAMING REPLICATION and REPMGR

Configure the master and standby databases for Streaming Replication

For this example we will use this configuration :

Master :   dbddlx05 (192.88.68.32)

Slave    :   dbddlx06 (192.88.68.33)

IP virt : 192.88.68.30

Configuring network authorization on the master

Edit the file pg_hba.conf by adding this line :

local   replication     repmgr                                 trust
host    replication     repmgr         192.88.68.0/32          trust
host    replication     repmgr         127.0.0.1/32            trust
local   repmgr          repmgr                                 trust
host    repmgr          repmgr         192.88.68.0/32          trust
host    repmgr          repmgr         127.0.0.1/32            trust

Configuring the master

Edit the postgresql.conf of the master and add/uncomment those lines :

# address for the listener
listen_addresses = 'localhost,192.88.68.32'
# wal generation for hot standby
wal_level = 'hot_standby'

wal_log_hints = on

# activate the archive process for sending the WAL to the standby in case of a big lag
archive_mode = on

archive_command = 'test -f %p && scp -p %p dbddlx06:/bases/pgsql/<DB_NAME>/archives/%f '

max_wal_senders = 10
# activate the hot_standby
hot_standby = on

# number max of the replication slots used by repmgr, normally >= nb slaves

max_replication_slots = 5

=> restart the master

Creation of the user and the databas for REPMGR

On the master database, create the user and his database dedicated to the repmgr process

postgres@dbddlx05:~$ createuser -s repmgr
postgres@dbddlx05:~$ createdb repmgr -O repmgr

Changing the search_path of the user for using a specific schema :

postgres=# alter user repmgr set search_path to repmgr_bdd, "$user", public;
ALTER ROLE

Creating the master REPMGR configuration file :

create a file in /etc/repmgr/repmgr.conf :

# attention no uppercase in the cluster name
cluster=bdd
node=1
node_name=dbddlx05
conninfo='host=dbddlx05 user=repmgr dbname=repmgr'

# repmgr will create automatically replication slots for managing wal retention

use_replication_slots=1

Creating the standby database


Creating the standby REPMGR configuration file :

create a file in /etc/repmgr/repmgr.conf :

cluster=bdd
node=2
node_name=dbddlx06
conninfo='host=dbddlx06 user=repmgr dbname=repmgr'

# repmgr will create automatically replication slots for managing wal retention

use_replication_slots=1

Automatic failover


Modification of configuration file on all the nodes

You have to add in the sudo configuration the possiblity to postgres user to manage postgesql with systemctl command

postgres@dbddlx06:/etc/postgresql/9.5/main$ cat  /etc/repmgr/repmgr.conf
cluster=bdd
node=2
node_name=dbddlx06
conninfo='host=dbddlx06 user=repmgr dbname=repmgr'
pg_bindir=/usr/lib/postgresql/9.5/bin

loglevel=INFO
logfile='/var/log/postgresql/repmgr.log'

start_command='sudo systemctl start postgresql@9.5-main.service'
stop_command='sudo systemctl stop postgresql@9.5-main.service'
restart_command='sudo systemctl restart postgresql@9.5-main.service'

# Automatic failover
failover=automatic
promote_command='repmgr standby promote -f /etc/repmgr/repmgr.conf'
follow_command='repmgr standby follow -f /etc/repmgr/repmgr.conf'
master_response_timeout=20

Starting the REPMGRD daemon on all the standby


postgres@dbddlx06:/etc/postgresql/9.5/main$ repmgrd -f /etc/repmgr/repmgr.conf -p /var/run/postgresql/repmgrd.pid -d --verbose >>/var/log/postgresql/repmgr.log 2>&1
